# iOS シミュレーターでのスクリーンショット撮影手順

## 前提条件

- Xcode がインストールされていること（Mac App Store から無料ダウンロード）
- プロジェクトの依存関係インストール済み（`npm install` 完了）

## 手順

### 1. 必要なシミュレーターを追加する

```bash
# 利用可能なシミュレーターを確認
xcrun simctl list devices
```

App Store 必須サイズ用に以下のシミュレーターを用意する：

| 優先度 | デバイス名 | 解像度 |
|--------|----------|--------|
| 必須 | iPhone 16 Pro Max | 1320 × 2868 |
| 推奨 | iPhone 16 Plus | 1290 × 2796 |
| 推奨 | iPhone 11 Pro Max | 1242 × 2688 |

Xcode > Window > Devices and Simulators > Simulators タブから追加できる。

### 2. アプリをシミュレーターで起動する

```bash
cd /Users/toshiki-kojima/my-project/kashikari-me-beta
npm run ios
```

または特定のシミュレーターを指定して起動：

```bash
npx expo start --ios --no-dev
```

Metro バンドラーが起動したら、Expo CLI の `i` キーを押して iPhone シミュレーターを選択。

### 3. シミュレーターのサイズを変更する（表示スケール調整）

Xcode シミュレーターで：
- **Window > Device Type** から対象デバイスに切り替え

### 4. スクリーンショットを撮影する

#### 方法A: シミュレーターのメニューから撮影（推奨）

1. 撮影したい画面まで操作する
2. シミュレーターのメニューバー: **File > Save Screen Shot**（または `Cmd + S`）
3. デスクトップに PNG で保存される

#### 方法B: xcrun コマンドで撮影

```bash
# デバイスID を確認
xcrun simctl list devices | grep Booted

# スクリーンショット撮影（<device-id> を実際のIDに置き換える）
xcrun simctl io <device-id> screenshot ~/Desktop/kashikari-screenshot-01.png
```

#### 方法C: Mac のスクリーンショット機能

- `Cmd + Shift + 4` → スペースキー → シミュレーターウィンドウをクリック
- ただしシミュレーターのフレーム（外枠）も写り込む

### 5. 撮影後の確認

- 解像度が要件通りかを確認（Finder でファイルを選択して `Cmd + I`）
- 必要に応じて Figma・Canva 等でキャプション・端末フレームを追加

### 6. ファイル命名規則

```
kashikari-<シーン番号>-<デバイス>.png

例:
kashikari-01-home-iphone16promax.png
kashikari-02-new-group-iphone16promax.png
kashikari-03-add-payment-iphone16promax.png
kashikari-04-payment-list-iphone16promax.png
kashikari-05-settlement-iphone16promax.png
kashikari-06-settlement-result-iphone16promax.png
```

撮影後は `/docs/release/screenshots/` 以下に配置する。

## 推奨撮影シーン（5〜6画面）

以下の順番でスクリーンショットを撮影することを推奨する。
ユーザーが「このアプリでできること」を一目で把握できる構成。

### Scene 1: グループ一覧画面（ホーム）

**目的:** アプリの全体像・複数グループを管理できることを伝える

**準備:**
- グループを2〜3件作成（例:「北海道旅行 🗾」「渋谷飲み会」「ルームシェア」）
- 各グループにメンバーを追加・支払い数件を記録しておく
- グループカードに総支出・メンバー数が表示された状態にする

**撮影ポイント:** カード一覧が見えること。カラフルで賑やかな印象を出す。

---

### Scene 2: グループ作成画面

**目的:** アカウント不要で名前を入力するだけで始められる手軽さを伝える

**準備:**
- 「新規グループ」をタップしてグループ作成画面を開く
- グループ名を入力済み状態（例:「沖縄旅行 ✈️」）
- メンバーを3〜4人追加した状態（例: たろう、はなこ、じろう、みどり）

**撮影ポイント:** 入力済みで今にも保存できる状態がわかりやすい。

---

### Scene 3: 支払い追加画面

**目的:** 「誰が・いくら・何に・誰の分か」を入力する主要操作を伝える

**準備:**
- 支払い追加画面を開く
- 金額: `¥12,800`
- 用途: `夕食（焼肉）`
- 支払った人: たろう
- カテゴリ: 食事
- 割り勘対象: 全員選択

**撮影ポイント:** 入力フォームが充実している印象を与える。

---

### Scene 4: 支払い一覧画面（グループ詳細・支払いタブ）

**目的:** 複数の支払い記録が時系列で並ぶ「記録帳」のイメージを伝える

**準備:**
- 5〜7件の支払いを記録（金額・用途・カテゴリのバリエーションを出す）
- 上部に総支出サマリー（例: `総支出 ¥48,600`）が表示された状態

**撮影ポイント:** リストが見え、総支出が目に入る構成。

---

### Scene 5: 精算タブ（残高一覧）

**目的:** 各メンバーの残高が一目でわかることを伝える

**準備:**
- 精算タブに切り替える
- プラス（受け取る）・マイナス（払う）のメンバーが混在する状態
- 例: たろう +¥8,200 / はなこ -¥3,100 / じろう -¥5,100

**撮影ポイント:** 色分け（プラス/マイナス）で視覚的にわかりやすい状態。

---

### Scene 6: 精算案（送金リスト）

**目的:** 「誰が誰にいくら払えばよいか」が自動計算されることを伝える（アプリの核心価値）

**準備:**
- 精算案が「はなこ → たろう ¥3,100」「じろう → たろう ¥5,100」のように表示された状態
- 精算済みチェックがついている行と未完了の行が混在している状態が理想

**撮影ポイント:** 「自動計算」「最小回数」の価値が伝わる。このシーンを先頭（Scene 1）に移動することも検討。

---

## ヒント

- シミュレーターの言語・地域設定を「日本語／日本」にしてから撮影する（設定 > 一般 > 言語と地域）
- ステータスバーの時刻は `9:41 AM` に設定するとプロフェッショナルな印象になる（Apple の標準ショット時刻）
  ```bash
  # シミュレーターのステータスバー時刻を設定
  xcrun simctl status_bar <device-id> override --time "9:41"
  ```
- 撮影後は App Store Connect に直接アップロードするか、Fastlane Deliver でメタデータと一括アップロードできる
